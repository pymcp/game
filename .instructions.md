# Mining Game - Development Instructions

## Project Overview
This is a 2-player split-screen mining game built with Python and Pygame. Players explore a shared world, collect resources, mine materials, fight enemies, and level up together.

## Architecture Summary

### Key Files & Structure
```
src/
├── game.py                 # Main game loop, split-screen rendering, and orchestration
├── config.py              # Game constants (screen size, world dimensions, tile sizes)
├── entities/
│   ├── player.py         # Player class with movement, mining, combat
│   ├── enemy.py          # Enemy AI with data-driven rendering
│   ├── projectile.py     # Weapon projectiles
│   ├── worker.py         # Resource gathering NPCs
│   └── pet.py            # Pet followers
├── world/
│   ├── generation.py     # Procedural world generation
│   └── collision.py      # Collision and blocking detection
├── data/
│   ├── tiles.py          # Tile definitions (grass, dirt, ore, etc.)
│   ├── enemies.py        # Enemy type specifications
│   ├── weapons.py        # Weapon data
│   └── pickaxes.py       # Tool upgrade specifications
├── effects/
│   ├── particle.py       # Visual effects (dust, etc.)
│   └── floating_text.py  # Floating damage/XP text
└── ui/
    └── hud.py            # HUD drawing functions
```

## Current Implementation Status

### ✅ Core Features Implemented
1. **Split-Screen 2-Player Mode**
   - Independent players with separate input controls
   - Shared world visible from both viewpoints
   - Both players appear in each other's viewports

2. **Player Controls**
   - **Player 1 (Left Viewport)**: WASD to move, F/RMB to fire, U/N/B for upgrades
   - **Player 2 (Right Viewport)**: Numpad 8/4/6/2 to move, KP_Enter to fire, KP_*/−/+ for upgrades

3. **Shared World System**
   - Single world shared between both players
   - All entities (enemies, NPCs) shared
   - Resources and enemy targeting work for both players

4. **Combat & Enemies**
   - Enemies target the closest player
   - Both players can damage enemies
   - XP split between players on enemy kill

5. **Window Management**
   - Resizable window (drag edges)
   - Fullscreen toggle (F11)
   - Game scales dynamically to fill window

6. **Border/World Edge**
   - Stone brick decorative border at world edges
   - Camera clamping prevents viewing outside world bounds
   - Prevents screen bleed

### 🔧 Key Implementation Details

#### Split-Screen Rendering
- `draw()` method calculates viewport dimensions from current window size
- Each viewport gets half the width, full height
- `_draw_player_view()` renders terrain, entities, and UI for one viewport
- Both players drawn in both viewports (not just their own)

#### Shared World Logic
- Single `world` grid shared by both Game, players, and entities
- Enemies use `_update_enemies()` which has each enemy target closest player
- Projectiles split XP between both players
- Workers randomly assign resources to either player
- Pets follow the closest player

#### Camera System
- `cam1_x/y` and `cam2_x/y` - independent cameras
- Smooth follow with 0.1 interpolation factor
- Clamped to world bounds: `max(0, min(cam_x, world_pixel_w - viewport_w))`

#### Dynamic UI
- UI shows stats for the viewport's primary player only
- Health, XP, inventory, weapon level displayed per-player
- Font size: 24px for stats

### 📋 Important Constants
- **Screen**: 960×640 pixels (resizable)
- **Viewport**: 480×full_height (split)
- **Tile Size**: 32×32 pixels
- **World**: 80×60 tiles (2560×1920 pixels)
- **FPS**: 60

## Common Patterns & Conventions

### Coordinate System
- World coordinates: absolute (x, y) in pixels
- Screen coordinates: relative to viewport (screen_x, screen_y)
- Camera offset: `screen_x = world_x - cam_x + viewport_screen_x`

### Entity Updates
- All entities use `update(dt, ...)` signature
- `dt` is delta time in milliseconds (~16.667 at 60 FPS)
- `dt` comes from: `self.clock.tick(FPS) / 16.667`

## Quick Debug Tips
1. **Game won't start**: Check pygame import with `python3 -c "import pygame"`
2. **Screen is black**: Check border rendering logic, may be filling entire screen
3. **Players not visible**: Check both `_draw_player_view()` calls in `draw()`
4. **Enemies not spawning**: Check `spawn_enemies()` in world.py
5. **Viewport bleed**: Verify camera clamping in `update()` method
